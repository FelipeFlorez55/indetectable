# 09 — Internationalization (i18n)

> New requirement. Owns: a tiny locale layer used by `src/components/` and
> [07-share-card](./07-share-card.md). **No dependency** — a ~50-line custom
> solution honors CLAUDE.md §3 ("minimize dependencies").

## Goal

Ship the UI in **three languages — Spanish (`es`), English (`en`), Portuguese
(`pt`)** — auto-detected from the browser, with a manual override. **Translation
touches display strings ONLY.** It must never affect game logic.

## The hard rule: i18n is presentation, logic is English-only

The locale changes what the player *reads*, nothing else. The following stay in
**English / locale-independent** so the daily challenge is identical worldwide
and the detector/persistence are deterministic:

- `seedKey(dayIndex)` = `"indetectable-<dayIndex>"` — **never localized** (else the
  daily scene would differ per language → breaks fairness & sharing, CLAUDE.md §8).
- `STORAGE_KEY`, `SCHEMA_VERSION`, export/import blob format (06).
- `FeatureKey` values and all detector internals (03).
- `dayIndex`, `level`, scores, streak math — pure numbers.

> Acceptance hinge: switching language must produce a **byte-identical** `Scene`,
> `Verdict`, and persisted state. Only rendered text changes.

## Detection & selection

```ts
export type Lang = "es" | "en" | "pt";
export const SUPPORTED: Lang[] = ["es", "en", "pt"];
export const DEFAULT_LANG: Lang = "en"; // fallback when nothing matches

/** Pick the best supported language from navigator.languages (pure; inject for tests). */
export function detectLang(preferred?: readonly string[]): Lang;

/** Persisted manual override (own localStorage key, separate from game state). */
export const LANG_STORAGE_KEY = "indetectable:lang";
export function loadLangOverride(storage?: Storage): Lang | null;
export function saveLangOverride(lang: Lang, storage?: Storage): void;

/** Resolution order: manual override → browser detection → DEFAULT_LANG. */
export function resolveLang(storage?: Storage, preferred?: readonly string[]): Lang;
```

- `detectLang` matches on the primary subtag (`pt-BR` → `pt`, `es-419` → `es`).
- Override persists in its **own** key, never inside `GameState` (06) — keeps the
  export/import blob locale-free.

## Dictionary & lookup

```ts
/** Stable, English-named keys. Values are the translated strings. */
export type MessageKey =
  | "app.title"
  | "intro.prompt"
  | "verdict.undetectable"   // "Undetectable 🫥"
  | "verdict.caught"         // "Caught you in {ms}"
  | "share.cta"
  | "share.whatsapp"
  | "stats.streak"
  | /* …complete set defined during impl… */ string;

export type Dictionary = Record<MessageKey, string>;
export const MESSAGES: Record<Lang, Dictionary>;

/** Translate with optional interpolation: t("verdict.caught", { ms: "0.3s" }). */
export function makeT(lang: Lang): (key: MessageKey, vars?: Record<string, string | number>) => string;
```

- Missing key in a non-default locale → fall back to `en` value (and warn in dev).
- Interpolation tokens are `{name}`; no logic in templates.
- Verdict flavor text ("Undetectable", "Caught you …") is localized; the share
  **URL** and **day number** are not (07).

## React wiring

```ts
// Minimal context; no external i18n library.
export const LangProvider: React.FC<{ children: React.ReactNode }>;
export function useT(): { lang: Lang; t: ReturnType<typeof makeT>; setLang: (l: Lang) => void };
```

- `LangProvider` resolves the language on mount (`resolveLang`) and exposes a
  setter that persists the override and re-renders.
- A language switcher lives in the Header / Stats screen (08).
- `<html lang>` is updated to the active locale for accessibility/SEO.

## Acceptance criteria

- [ ] `detectLang` maps `es*/en*/pt*` correctly and falls back to `en` for
      anything else — pure, unit-tested with injected language lists.
- [ ] Switching language changes only rendered text; `Scene`, `Verdict`, and the
      persisted/export blob are **byte-identical** across locales (asserted test).
- [ ] `seedKey`, `STORAGE_KEY`, `FeatureKey`, and export format contain **no**
      localized text — guarded by a test grepping for the constant `"indetectable-"`.
- [ ] Every `MessageKey` exists in all three dictionaries (completeness test);
      a missing key falls back to `en` without throwing.
- [ ] Manual override persists across reloads in its own storage key, separate
      from `GameState`.
- [ ] All three locales (es/en/pt) have human-reviewed copy for verdicts and
      share text before release.

## Out of scope

- RTL languages, locale-aware number/date formatting beyond simple cases, and
  lazy-loading dictionaries (the three are tiny — bundle them).
